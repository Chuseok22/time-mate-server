### 📌 작업 개요

`DELETE /api/users/me` 회원탈퇴 API 구현. Hard Delete 방식으로 사용자 계정과 관련 참가자 기록 전체 삭제.

### 🎯 구현 목표

로그인 사용자가 본인 계정을 완전 삭제할 수 있는 API 제공.

### ✅ 구현 내용

#### UserRepository — deleteById 추가
- **파일**: `user/core/repository/UserRepository.java`, `user/application/repository/UserRepositoryImpl.java`
- **변경 내용**: `deleteById(UUID)` 계약 정의 및 `existsById` 선행 검증 후 JPA 위임 구현
- **이유**: 기존 MeetingRoomRepositoryImpl TOCTOU 방지 패턴 동일하게 적용

#### UserService — deleteUser 추가
- **파일**: `user/core/service/UserService.java`, `user/application/service/UserServiceImpl.java`
- **변경 내용**: `deleteUser(UUID)` 구현. `AvailabilityTimeRepository` 필드 추가
- **이유**: FK 순서(AvailabilityTime → Participant → User) 준수를 위해 서비스 계층에서 순서 제어

#### UserController — DELETE /api/users/me 추가
- **파일**: `user/application/controller/UserController.java`
- **변경 내용**: JWT에서 userId 추출 후 `userService.deleteUser(userId)` 호출, 204 반환
- **이유**: `/api/users/**`는 SecurityConfig에서 이미 `.authenticated()` 처리되어 별도 설정 불필요

#### UserControllerDocs — Swagger 문서 추가
- **파일**: `user/application/controller/docs/UserControllerDocs.java`
- **변경 내용**: 동작 과정, 유의사항(MeetingRoom 잔존, 복구 불가) 포함한 @Operation 추가

### 🔧 주요 변경사항 상세

**삭제 순서 (Hard Delete)**
```
1. userRepository.findById(userId)       ← 존재 확인, 없으면 USER_NOT_FOUND
2. participantRepository.findAllByUserId → Participant 목록 조회
3. availabilityTimeRepository.deleteAllByParticipant(each)  ← FK 선행 삭제
4. participantRepository.deleteById(each.getId())
5. userRepository.deleteById(userId)     ← User 삭제
```

**MeetingRoom 처리 정책**: `MeetingRoom.creatorUserId`는 FK 없는 UUID 컬럼이므로 User 삭제 후에도 row 유지. 방 자체는 다른 참가자 공유 데이터이므로 보존. 단, 탈퇴한 사용자의 방은 이후 삭제 불가.

### 🧪 검증 결과

- `./gradlew compileJava` → BUILD SUCCESSFUL
- `./gradlew cleanTest test` → BUILD SUCCESSFUL (28/28 PASS)

추가된 테스트:
- `deleteUser_withParticipants_deletesInOrder` — 참가자 있을 때 AvailabilityTime → Participant → User 순 삭제 검증
- `deleteUser_noParticipants_deletesUserOnly` — 참가자 없을 때 User만 삭제, 불필요한 repository 호출 없음 검증
- `deleteUser_userNotFound_throwsCustomException` — 존재하지 않는 userId 시 USER_NOT_FOUND, deleteById 미호출 검증

### ⚠️ 위험 요소

- 탈퇴 사용자가 생성한 MeetingRoom은 `creatorUserId`가 남아 있으나 실제 User 없음. 해당 방의 삭제 불가 상태가 됨. 별도 처리 필요시 후속 이슈로 관리 권장.
